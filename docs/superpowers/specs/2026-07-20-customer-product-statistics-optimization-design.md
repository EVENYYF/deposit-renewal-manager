# 客户、产品与统计体验优化设计

日期：2026-07-20

## 1. 背景与目标

本轮优化处理五类体验问题：客户卡片缺少独立资料页、修改记录暴露内部字段名、底部导航项目过多、产品和利率缺少结构化管理，以及四种存款状态无法查看明细。

本设计继续使用现有 Flutter、Riverpod、Drift、UseCase 和 Repository 分层。数据库升级到 schema v7，新增结构化产品与利率版本，保留现有存款快照语义，不引入云同步、在线账号或全局缓存框架。

## 2. 导航结构

底部 NavigationBar 和桌面 NavigationRail 从五项缩减为四项：

```text
首页 | 客户 | 新增 | 设置
```

原“模板”分支移除，模板管理作为设置页入口打开。设置页包含：

- 消息模板
- 产品管理
- 存款统计
- 备份、恢复与快照
- 系统通知设置
- 系统限制说明

模板页面本身及其绑定保持不变，只调整入口和路由归属。手机和桌面导航使用相同的四项顺序。

## 3. 客户详情页

### 3.1 页面结构

客户列表卡片不再展开存款列表。点击卡片进入独立客户详情页，页面包含：

1. 客户资料区域：姓名、手机号、编辑按钮。
2. 手机号操作：未填写时显示“添加手机号”；已填写时显示复制按钮。
3. 存款区域：展示该客户全部存款链和生命周期。
4. 修改记录入口。
5. 新增存款入口。

客户列表仍展示姓名、手机号摘要和存款数量，并保留搜索、银行筛选、产品筛选和下拉刷新。

### 3.2 编辑与复制

客户详情复用现有客户编辑表单，支持修改姓名、添加手机号和修改手机号。保存成功后刷新客户控制器，并用新的 `CustomerSearchResult` 更新详情页和存款链。

手机号存在时可复制到剪贴板，成功显示“已复制手机号”。手机号为空时不显示复制按钮。复制失败保留当前页面并显示“复制失败，请重试”。

## 4. 修改记录展示

审计数据继续读取现有 `audit_history`，展示层增加统一字段规范化和格式化。

### 4.1 字段名称

同时兼容蛇形和驼峰字段：

```text
name                                      姓名
phone                                     手机号
bank_name / bankName                      银行
product_name / productName                产品
amount_cents / amountCents                金额
interest_rate_scaled / interestRateScaled 年利率
rate_precision / ratePrecision            利率精度
term_value / termValue                    期限
term_unit / termUnit                      期限单位
start_date / startDate                    存入日期
calculated_expiry_date / calculatedExpiryDate 计算到期日
final_expiry_date / finalExpiryDate        到期日
lifecycle                                  状态
```

未知字段转换为空格分隔的可读名称，不直接暴露未处理的内部标识。

### 4.2 值格式

- 金额从分转换为元并使用货币格式。
- 利率结合 `ratePrecision` 转换为百分比。
- `active`、`renewed`、`stopped` 转换为“生效中”“已续期”“已停止”。
- `day`、`month`、`year` 转换为“天”“个月”“年”。
- 日期统一显示为 `yyyy-MM-dd`。
- 空值显示为“未填写”。

### 4.3 颜色

- 字段名称使用主题强调色。
- 原值使用弱化灰色；被替换或移除的值可使用错误色。
- 新值使用成功绿色。
- 每个变更独立换行，避免长 JSON 字段挤在同一行。

## 5. 产品与利率领域模型

### 5.1 数据表

schema v7 新增两张业务表。

`products`：

```text
id                  TEXT PRIMARY KEY
bank_name           TEXT NOT NULL
product_name        TEXT NOT NULL
is_active           INTEGER NOT NULL DEFAULT 1
created_at_utc      INTEGER NOT NULL
updated_at_utc      INTEGER NOT NULL
```

唯一约束为标准化后的银行名和产品名组合。不同银行的同名产品是独立配置。

`product_rate_versions`：

```text
id                    TEXT PRIMARY KEY
product_id            TEXT NOT NULL REFERENCES products(id)
interest_rate_scaled  INTEGER NOT NULL
rate_precision        INTEGER NOT NULL
effective_date        TEXT NOT NULL
created_at_utc        INTEGER NOT NULL
updated_at_utc        INTEGER NOT NULL
```

同一产品和同一生效日期唯一。产品删除采用停用，不级联删除利率版本。

### 5.2 产品管理

设置页新增“产品管理”入口。页面支持：

- 按银行和产品名搜索。
- 新增产品。
- 编辑银行和产品名称。
- 停用和重新启用产品。
- 查看按生效日期倒序排列的利率版本。
- 新增或编辑利率版本。

利率必须大于或等于 0，生效日期必须是有效本地日期。同一产品同一生效日期再次保存时更新原版本，不创建重复记录。

历史存款保存银行、产品和利率快照，产品配置修改或停用不会改写已有存款。

### 5.3 与现有预设的关系

现有 `deposit_presets` 继续服务金额、银行、期限等候选值。产品和利率的主要选择来源改为新产品领域模型；旧产品和利率预设可继续保留，避免 schema 迁移期间丢失候选数据，但不承担产品与利率绑定关系。

## 6. 存款表单联动

新增和续期表单使用以下流程：

```text
选择银行
  → 加载该银行下启用产品
  → 选择产品
  → 输入或选择存入日期
  → 查找 effective_date <= start_date 的最新利率版本
  → 自动填入年利率
  → 用户仍可手动覆盖
```

银行、产品或存入日期变化时重新匹配。若没有适用版本，保留已有手工值或空值并显示提示，不阻止用户继续填写和保存。

编辑已有存款时优先展示存款自身快照，不因产品配置变化自动覆盖。续期表单可以根据新的存入日期匹配当前适用利率。

## 7. 存款状态统计钻取

### 7.1 状态定义

- 生效中：`lifecycle = active` 且 `final_expiry_date >= today`。
- 已逾期：`lifecycle = active` 且 `final_expiry_date < today`。
- 已续期历史：`lifecycle = renewed`。
- 已停止：`lifecycle = stopped`。

四个状态行均可点击，并显示进入箭头。

### 7.2 详情查询

统计详情查询扩展为状态和分类两类过滤：

```text
status: active | overdue | renewed | stopped
dimension: bank | product
```

详情列表展示客户姓名、手机号、银行、产品、金额、年利率、存入日期、到期日期和状态。手机号存在时支持复制。

列表按到期日、客户姓名和存款 ID 稳定排序。查询继续排除已停用客户，但状态查询不再错误套用“仅 active”条件。

点击单笔记录打开统一存款详情。活动存款显示续期、停止续期和编辑操作，历史记录只读。

## 8. 数据库迁移与备份

### 8.1 schema v6 到 v7

升级事务创建 `products`、`product_rate_versions`、唯一索引和外键索引。迁移不从历史存款自动生成产品配置，避免把用户录入差异错误合并为正式产品。

### 8.2 备份兼容

- v7 备份包含产品、利率版本和停用状态。
- 恢复影响预览计算新表中将被替换或删除的记录。
- v7 往返恢复保持产品和利率版本完整。
- 继续接受 v3、v4、v5、v6 备份。
- 旧备份标准化时为两张新表补充空列表，恢复后由用户建立产品配置。
- 备份清单、数据结构校验和原子替换顺序同步更新，产品先于利率版本写入。

## 9. 组件与职责

- `CustomerDirectoryPage`：搜索、筛选和客户摘要列表。
- `CustomerDetailPage`：客户资料、存款链和修改记录入口。
- 共享存款详情组件：客户详情和统计详情复用生命周期操作。
- `ProductRepository` / `ProductRateRepository`：产品和利率版本持久化。
- `ProductManagementPage`：产品配置管理。
- `DepositFormPage`：消费产品查询和按日期匹配服务。
- `DepositStatisticsUseCases`：支持状态和银行/产品详情过滤。
- 设置路由：承载模板、产品、统计、备份和通知入口。

## 10. 错误处理

- 客户保存失败时保留编辑内容并显示可读提示。
- 复制手机号失败不离开页面。
- 产品重名、同日利率重复和输入格式错误在表单层反馈。
- 并发修改导致记录失效时刷新列表并提示重试。
- 自动匹配利率失败不阻止手工填写。
- 统计和产品列表加载失败保留重试入口。
- 通用错误不向用户展示数据库 SQL 或内部堆栈。

## 11. 测试与验证

新增或更新以下测试：

1. 客户卡片进入详情，编辑姓名和手机号，复制手机号。
2. 客户详情刷新后同步资料和存款链。
3. 修改记录字段中英文映射、金额、利率、期限、日期和状态格式。
4. 原值、新值和字段标签颜色。
5. 手机和桌面导航均为四项，模板可从设置进入。
6. schema v6 到 v7 迁移、新表约束和外键。
7. 产品新增、编辑、停用、启用及多利率版本。
8. 银行、产品和存入日期变化时自动匹配利率，手动覆盖不被意外重置。
9. v3-v7 备份兼容、v7 往返恢复和恢复影响预览。
10. 四种状态统计边界和详情列表。
11. 统计详情单笔记录进入统一存款详情。
12. 运行 Dart 格式检查、`flutter analyze`、全量 `flutter test` 和 Android Debug APK 构建。

真机验收包括客户资料复制、产品利率匹配、四状态钻取和四项底部导航。设备连接不顺畅时最多尝试三次，之后交由用户手动验证。

## 12. 完成标准

- 客户列表点击进入独立详情页，可编辑资料并复制手机号。
- 修改记录不再直接显示 `amountCents` 等内部字段名。
- 底部导航缩减为四项，模板和产品管理均位于设置。
- 产品按银行区分，支持多个按日期生效的利率版本和停用状态。
- 新增与续期表单可按存入日期自动匹配利率且允许覆盖。
- 四种存款状态均可查看明细并进入单笔详情。
- schema v7 迁移和旧备份兼容测试通过。
- 静态分析、全量测试和 Debug APK 构建通过。
