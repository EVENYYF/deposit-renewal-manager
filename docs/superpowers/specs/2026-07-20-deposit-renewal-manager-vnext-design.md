# 存款续期管理工具 vNext 设计规格

## 目标

按测试清单修复 Android 通知与备份缺陷，并分层升级存款录入、续期历史、客户详情、统计和自动快照，同时保持离线 SQLite 架构与旧备份兼容。

## 实施分层

1. 通知与 Android 备份：首次申请通知权限；初始化失败可重试；系统设置入口有反馈；Android 使用 `FilePicker.saveFile(bytes:)` 导出归档。
2. 存款表单：客户姓名/手机号选择、日期选择器、自动到期日、产品与字段预设、续期确认弹窗。
3. 数据与客户管理：存款产品字段、同名同手机号提示、续期链展示、详情与状态样式、审计前后差异、银行/产品筛选。
4. 统计与快照：存款统计页面；新增设备本地快照策略，不覆盖跨设备业务设置。

## 数据边界

- 续期继续复用现有 `renewals` 链；新旧每期利率、产品和期限独立保存。
- 同名同手机号只弹窗提示，不增加数据库唯一约束。
- 自动快照策略属于设备本地数据，不进入业务备份；恢复前和批量导入前快照始终强制执行。
- schema v5 必须兼容现有 v4 数据库及 v3/v4 备份。

## 验收

- `flutter analyze`、完整 `flutter test`、Windows 集成测试通过。
- Android Release 构建成功。
- Android 真机验证通知权限、初始化重试、系统设置和备份导出；未实测项目明确标记为待验证。
- 迁移、备份往返、续期链、审计差异、统计去重和快照保留均有自动化测试。
