# 存款续期管理工具设计说明

## 1. 项目目标

为个人用户提供一套以 Android 手机为主、Windows 电脑为辅的离线存款续期管理工具。系统用于记录客户及其多笔存款，在应用内展示到期客户，并通过 Android 系统通知主动提醒。手机与电脑均可离线编辑，通过备份文件手动交接完整数据。

首版不依赖服务器、不要求登录、不调用在线 AI，也不自动发送短信或微信消息。

## 2. 已确认约束

- 平台：Android 与 Windows。
- 主用设备：Android 手机。
- 数据存储：每台设备使用本地 SQLite 数据库。
- 同步方式：手动导出、导入完整备份；不做自动同步或记录级合并。
- 通知方式：每日汇总通知与单笔关键节点通知并存。
- 文字识别：完全离线，识别结果必须人工确认。
- 到期日：每笔记录可选择自动计算或直接填写。
- 安全：首版不提供应用锁、数据库加密和备份加密。
- 电脑端：可查看并编辑全部数据。

## 3. 技术架构

采用 Flutter 构建 Android 与 Windows 应用，共享领域模型、业务规则、数据访问与大部分界面组件。

```text
Flutter 应用
├── 展示层
│   ├── Android：底部导航与单列操作
│   └── Windows：侧边导航与宽屏表格
├── 业务层
│   ├── 客户与存款管理
│   ├── 到期日计算
│   ├── 续期与状态流转
│   ├── 提醒计划
│   ├── 离线文字解析
│   ├── 搜索与拼音索引
│   └── 提示语生成
├── 数据层
│   ├── SQLite 本地数据库
│   ├── 修改历史与导入批次
│   └── 整库备份导入/导出
└── 平台层
    ├── Android：本地通知、重启恢复、文件分享
    └── Windows：文件选择与导入导出
```

首版不引入私人服务器。未来如增加同步服务，不改变本地数据库作为可独立运行数据源的原则。

## 4. 核心数据模型

### 4.1 客户

- 客户 ID
- 姓名
- 姓名全拼与拼音首字母索引
- 手机号
- 备注
- 状态：正常、停用
- 创建时间、更新时间

一个客户可以关联多笔存款。客户存在存款或历史记录时不做物理删除，改为停用。

### 4.2 存款

- 存款 ID、客户 ID
- 金额
- 银行与产品名称
- 利率
- 存入日期
- 存期数值与单位（日、月、年）
- 到期模式：自动计算、直接填写
- 计算到期日、最终到期日
- 是否手工调整到期日
- 生命周期状态：生效中、已续期、已停止
- 备注
- 创建时间、更新时间

### 4.3 续期关系

- 原存款 ID
- 新存款 ID
- 续期操作时间

续期时原存款标记为已续期，并创建新的生效存款。旧记录不被覆盖。

### 4.4 修改历史

记录客户、存款关键字段的修改前值、修改后值、操作时间和来源设备。首版提供查看，不提供任意版本回滚。

### 4.5 提示语模板

包含模板名称、正文、启用状态与默认标记。正文支持客户姓名、金额、银行、产品、利率、存入日、到期日等变量。

“已到期”是展示状态：生命周期为生效中且最终到期日早于本地今日时，由查询即时派生，不依赖应用在午夜执行后台任务。

### 4.6 导入批次

记录 Excel 导入时间、来源文件名、成功数量、跳过数量、失败原因摘要和受影响记录，用于审计并撤销最近一次导入。

### 4.7 应用设置

分为两类：

- 可备份业务设置：每日汇总时间、单笔提前提醒天数、默认提示语模板。
- 设备本地设置：设备名称和平台通知能力，不随备份导入而覆盖。通知权限每次从系统读取，不作为数据库中的可恢复状态。

## 5. 业务规则

### 5.1 到期日

- 自动模式根据存入日期、存期数值和单位计算。
- 日存期按自然日相加；月、年存期按日历月或日历年相加。
- 当目标月份不存在对应日期时，使用目标月份最后一天，例如 1 月 31 日加 1 个月得到 2 月最后一天。
- 闰年使用实际日历规则。
- 自动结果允许人工修正，修正后保留计算值，并标记最终日期已调整。
- 直接填写模式只使用用户指定的最终到期日。

### 5.2 状态

```text
生效中 ──续期──→ 已续期 ──关联──→ 新的生效存款
   │
   └──停止续期──→ 已停止
```

停止续期只终止该笔记录的后续提醒，不删除历史。状态变化后立即重新计算并安排通知。已到期是从生效状态与最终到期日派生的待处理状态，仍允许续期或停止续期，不提供“重新激活”操作。客户仍有生效存款时禁止停用，用户必须先逐笔续期、停止或处理相关存款。

### 5.3 首页提醒分组

三个分组互斥，同一笔存款只出现一次：

1. 今日到期。
2. 未来 3 天到期，即本地日期 `today + 1` 至 `today + 3`，即使跨周也归入本组。
3. 本周其余日期到期，不含前两个分组。

“本周”按本地时区的周一至周日计算。仅生效中的存款进入未来提醒；生命周期为生效中且最终到期日早于本地今日的记录即时显示为到期待处理。设备时区变化后，应用在下次启动或恢复前台时重新计算分组与通知。

### 5.4 通知

- 默认每天固定时间发送今日、未来 3 天、本周内和到期待处理的数量汇总。
- 单笔存款默认在到期前 7 天、3 天和到期当天发送通知。
- 提醒时间和提前天数可在设置中调整。
- 新增、修改、续期、停止续期或导入数据后重新安排受影响通知。
- 首版支持 Android 10 及以上。优先使用系统允许的精确调度；精确闹钟能力不可用时降级为系统非精确调度，并在设置页说明通知可能延迟。
- Android 设备重启、应用升级或系统时区变化后重新安排通知；应用启动和恢复前台时执行通知完整性校验，并补充显示已经错过的到期事项。
- 通知权限或必要的闹钟能力不可用时，首页持续显示可操作警告。
- 点击单笔通知打开对应客户和存款详情。

用户在系统设置中“强行停止”应用后，Android 不保证任何后台通知；用户重新打开应用后才恢复调度。厂商省电策略导致的延迟也无法完全消除，应用内提醒始终作为兜底。

Windows 首版只显示应用内提醒，不承诺应用关闭后的系统通知。

## 6. 功能与页面

### 6.1 Android 导航

底部导航包含：首页、客户、新增、模板、设置。

- 首页依次显示今日到期、三天内、本周内和到期待处理记录。
- 存款快捷操作包括生成联系提示语、续期、更新和停止续期。
- 客户详情展示基本资料、有效存款、历史存款与修改记录。
- 关键操作使用可见按钮与确认提示，不依赖隐藏手势。

### 6.2 Windows 导航

使用侧边导航与宽屏表格，功能与手机端一致。支持姓名、手机号、银行、到期日和状态组合筛选。首版不提供批量修改。

### 6.3 新增与离线文字解析

用户可以手工录入，也可以粘贴大段文字。离线解析器提取：

- 姓名、手机号
- 金额、银行、产品、利率
- 存入日期、到期日期、存期
- 备注中无法结构化的剩余文本

解析流程为：清洗文本、规则与词典识别、生成候选字段、标记缺失或冲突、人工确认、写入正式数据。系统不得跳过确认直接保存。

### 6.4 搜索

客户搜索支持：

- 姓名汉字包含匹配
- 手机号部分匹配
- 姓名拼音全拼，例如 `zhangsan`
- 拼音首字母，例如 `zs`

姓名或拼音查询使用不区分大小写的标准化索引。全拼支持包含匹配，首字母支持前缀和包含匹配。结果按完全匹配、前缀匹配、包含匹配排序；首版不做编辑距离或错别字纠正。以 1 万名客户为基准，手机端搜索响应时间的 P95 目标低于 200 毫秒。

### 6.5 提示语

选择客户、存款和模板后替换变量生成提示语。生成内容可临时编辑并一键复制。首版不自动调用短信、微信或其他外部发送接口。

## 7. Excel 批量导入

- 支持 `.xlsx`，不支持旧版 `.xls`。
- 提供标准模板，也允许将自定义表头映射到系统字段。
- 流程：选择文件、映射字段、校验预览、处理重复、确认导入。
- 校验金额、日期、手机号、期限和必填项；错误行必须修正或跳过。
- 系统按手机号识别疑似已有客户。
- 每条疑似重复记录可选择归入已有客户、新增独立客户或跳过。
- 支持将同一处理方式批量应用到多条记录。
- 归入已有客户时默认保留已有客户资料并新增存款；资料不一致时展示差异并由用户选择字段值。
- 整批导入在单个数据库事务中执行，写入异常时整体回滚。
- 导入前自动生成带批次 ID 的完整快照，成功后生成导入批次。
- 仅当导入后没有发生任何业务数据写入时，允许“撤销最近导入”；撤销实际执行导入前快照的整库恢复。
- 导入后若发生编辑、续期或建立新关联，禁止直接撤销并说明原因。用户仍可从快照管理中选择恢复导入前版本，但该操作会整库覆盖，必须展示此后将丢失的记录数量并再次确认。

## 8. 备份与手动同步

备份文件包含全部客户、存款、续期关系、修改历史、模板、导入批次、可备份业务设置、数据库版本、来源设备、生成时间和各类记录数量。设备名称、系统通知权限和平台调度状态不写入可恢复业务数据。

```text
源设备导出完整备份
        ↓
目标设备选择并校验文件
        ↓
展示来源、时间、版本、记录数量
        ↓
自动保存目标设备当前快照
        ↓
用户确认后整库替换
```

- Android 与 Windows 使用同一备份格式。
- 首版不做增量同步、自动合并或冲突解决。
- 两端都修改过时，由用户选择以哪份整库数据为准。
- 自动快照保存在应用私有数据目录，使用唯一时间戳和操作类型命名，默认保留最近 10 份；超过数量时只清理最旧的自动快照，不影响用户手工导出的备份。
- 设置页提供快照列表、来源说明、记录统计和恢复入口。恢复任意快照前再次保存当前快照，因此错误恢复仍可回退。
- 导入或恢复失败时保留原数据库，不能留下部分写入，并保留失败前快照。
- 导入成功后重新生成搜索索引并重排 Android 通知。
- 因首版不加密，导出界面明确提示备份包含敏感信息。

## 9. 异常处理

- 数据库写入、续期、Excel 导入和备份恢复均使用事务。
- 备份版本不兼容、文件损坏或校验失败时禁止覆盖当前数据。
- 日期或金额无法解析时保留原输入并要求用户修正。
- 通知调度失败时记录可读错误，并在首页和设置页提供重试。
- 文件导入、导出取消属于正常中止，不显示为数据错误。
- 客户停用、停止续期、导入覆盖等高影响操作均要求确认。

## 10. 测试策略

### 单元测试

- 月末、跨年、闰年及日/月/年存期计算。
- 今日、三天内、本周内边界与互斥分组。
- 状态流转、续期链路和停止续期。
- 模板变量替换与缺失字段处理。
- 离线文字规则与常见中文格式。
- 拼音全拼、首字母、包含匹配、排序与 1 万客户性能基线。

### 数据与集成测试

- SQLite 迁移、事务回滚和修改历史。
- `.xlsx` 字段映射、校验、重复处理，以及新建客户、复用客户、字段覆盖场景下的批次撤销。
- 导入后发生编辑或续期时禁止直接撤销，并可通过快照预览影响后执行整库恢复。
- Android 与 Windows 备份互导、快照恢复、错误覆盖回退、损坏文件和版本不兼容。
- 导入后搜索索引重建及通知重排。

### 平台测试

- Android 断网、应用关闭、设备重启、应用升级、时区变化、精确调度降级和通知权限关闭场景。
- Android 主流屏幕尺寸与 Windows 宽屏布局。
- 大数据量列表、搜索和导入时的响应速度。

## 11. MVP 验收标准

- Android 和 Windows 均可离线新增、编辑、查询客户及多笔存款。
- 自动与手工到期模式工作正确，允许修正自动结果。
- 首页三个提醒分组准确且互不重复。
- Android 每日汇总与单笔重点通知可配置并能在重启后恢复。
- 续期保留旧记录并创建新记录；停止续期后不再通知。
- 可按姓名、手机号、拼音全拼、拼音首字母及组合条件搜索。
- 大段文字可离线提取核心字段，并在确认后保存。
- 可通过模板生成提示语、临时编辑并复制。
- 可导入 `.xlsx`，预览错误，逐条处理疑似重复客户；无后续写入时可撤销最近导入，有后续写入时禁止直接撤销。
- Android 与 Windows 可互相导入完整备份，导入前自动保存当前快照，并能从快照列表恢复错误覆盖前的数据。
- 关键修改可在历史中追溯，事务失败不会产生半成品数据。

## 12. 明确不在首版范围

- 私人服务器、账号体系、自动同步和多人协作。
- 数据库加密、备份加密、PIN 或生物识别应用锁。
- 在线 AI、OCR、语音录入。
- 短信、微信或其他渠道自动发送。
- 利息收益计算、复杂金融产品规则与统计报表。
- `.xls` 文件导入、批量修改和任意历史版本回滚。

## 13. 风险与后续演进

- 客户姓名、电话和存款属于敏感信息。首版明文数据库与备份一旦被复制即可读取，界面必须持续提示该风险。后续应优先增加备份加密和应用锁。
- Android 厂商对后台任务和精确通知限制不同，需要真机覆盖测试，并以应用内提醒作为通知异常时的兜底。
- 整库交接简单可靠，但两端并行编辑会产生覆盖风险。导入前快照和清晰的来源信息是必要保护；自动合并留待后续专门设计。
- Excel 来源格式差异较大，字段映射与预览必须比自动猜测优先。
