# 存款表单目录联动与中文日期选择器设计

日期：2026-07-20

## 目标

优化存款表单的三个交互问题：

1. 银行和产品支持“可输入 + 可选择”的下拉交互。
2. 选择银行后，产品候选项来自该银行的产品目录；每个目录产品按当前存入日期匹配利率，并在候选项中展示。目录产品使用强调色，普通历史候选项保持默认色。
3. 日期选择器显示中文，日期值仍使用现有 `YYYY-MM-DD` 文本格式，保留手工输入能力。

## 方案

采用 Flutter `Autocomplete` 的可编辑下拉方案。银行和产品仍由原有
`TextEditingController` 保存，用户可以直接键入未收录的名称；点击候选项时写回控制器并触发现有联动逻辑。

产品目录只展示当前银行的活跃产品。对每个产品调用现有
`ProductCatalogService.matchRate(productId, startDate)`，匹配成功时在候选副标题展示
`年利率 x.xx%`；没有适用版本时展示“该日期无利率版本”。匹配结果按目录产品顺序缓存，银行或存入日期变化时失效并重新计算。

## 组件与数据流

### 存款表单

- 用 `_catalogBanks` 作为银行 `Autocomplete` 的候选源。
- 用 `_catalogProducts` 作为产品候选源，并以输入文本过滤；每个候选项同时携带
  `ProductRecord` 和当前日期匹配到的 `ProductRateVersion?`。
- 目录候选项的选中回调设置 `_selectedProductId`，随后调用已有利率匹配流程更新利率输入框。
- 手工输入不清空目录列表，只在输入变化时取消当前产品 ID，避免把自由文本误当成目录产品。
- 银行变化触发按银行加载产品；存入日期变化触发所有当前银行产品的利率匹配。
- 竞态请求使用现有 generation 计数，过期响应不得覆盖当前银行或日期状态。

### 应用本地化

- 在 `pubspec.yaml` 增加 SDK 依赖 `flutter_localizations`。
- `MaterialApp.router` 增加 `localizationsDelegates` 和 `supportedLocales: [Locale('zh', 'CN')]`。
- `showDatePicker` 显式传入 `locale: const Locale('zh', 'CN')`，确保独立 Widget 测试也得到中文界面。

## 错误与边界

- 产品目录加载失败时，银行和产品仍可手工填写，表单不可因目录故障阻塞。
- 当前日期没有利率版本时，不覆盖用户已有利率，并显示可读提示。
- 用户手工修改利率后，后续日期变化不自动覆盖该值；重新选择目录产品时允许重新匹配。
- 银行为空时不请求产品目录，产品候选只保留普通历史候选项。
- 目录同名产品按产品 ID 区分，显示银行过滤后的全部活跃记录。

## 测试计划

- Widget 测试：银行输入展示目录候选，产品候选展示匹配利率和强调色。
- Widget 测试：手工输入银行/产品仍可提交，目录请求失败仍不阻塞输入。
- Widget 测试：存入日期改变后产品候选利率更新；手工利率不会被日期变化覆盖。
- Widget 测试：日期选择器显示中文月份、按钮和辅助文本。
- 服务层测试：多个产品和多个日期版本匹配正确，未命中返回空。
- 回归验证：`flutter analyze --no-pub`、`flutter test --no-pub`、`flutter build apk --debug --no-pub`。

## 不在本次范围

- 不调整产品目录数据库结构、备份格式或设置页管理流程。
- 不改变存款数据保存格式和现有利率精度规则。
